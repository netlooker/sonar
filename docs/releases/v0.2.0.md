# Sonar v0.2.0

Release date: 2026-05-17

## Summary

Sonar v0.2.0 expands the high-value research path rather than just tuning the
existing surface.

This release adds:

- PDF extraction backed by `pymupdf`
- preserved PDF body text in prepared bundle `full_text`
- semantic topic-result filtering in `sonar_collect_sources_for_topic`
- embeddings configuration for topic pruning
- updated Sonar operator and skill documentation

## Highlights

### PDF extraction is now first-class

- Sonar now extracts `application/pdf` responses and direct `.pdf` URLs with
  `pymupdf`
- extracted PDF text is preserved in normal extraction records
- prepared paper/source bundles now carry PDF-derived content in `full_text`
  instead of degrading to metadata-only output

This closes an important gap in academic and document-heavy retrieval flows.

### Topic collection now prunes false positives semantically

- `sonar_collect_sources_for_topic` now over-collects paper candidates
- Sonar computes embeddings for the topic and candidate abstract/summary text
- low-similarity candidates are discarded before the final bundle is returned
  and persisted

This reduces off-topic paper selection in broad research queries.

### Config surface expanded for relevance filtering

- new `[embeddings]` settings for enablement, base URL, model, and similarity
  threshold
- provider auth can be supplied with `SONAR_EMBEDDINGS_API_KEY` or
  `OPENAI_API_KEY`
- if embeddings are unavailable, topic collection continues with a warning
  rather than hard-failing

## Downstream Impact

- downstream systems such as Synapse receive better prepared evidence for
  PDF-backed sources
- topic-based source collection is now more selective and less noisy
- operator guidance now matches the actual retrieval and filtering behavior

## Known Limitation

Topic filtering depends on a working embeddings provider configuration. Without
that configuration, Sonar still returns topic bundles, but semantic pruning is
skipped and the warning list should be treated as operationally significant.
