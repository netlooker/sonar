# Sonar v0.3.1

Release date: 2026-06-09

## Summary

Sonar v0.3.1 is an adoption and packaging patch release. It adds a verified
MCP-first onboarding path, corrects the default lightweight Docker build, and
reorganizes documentation around current user workflows.

## Highlights

- added a Compose `sonar-mcp` profile that starts Streamable HTTP MCP with its
  SearxNG dependency at `http://127.0.0.1:8000/mcp`
- added a current OpenCode remote MCP configuration example
- corrected untargeted Docker builds so they produce the lightweight API image
  instead of the optional browser runtime
- added an optional `Justfile` for setup, checks, local services, Compose, and
  OpenAPI generation
- added contributor guidance and repository-native documentation contract tests
- replaced overlapping and obsolete documents with task-oriented MCP, HTTP,
  resilient retrieval, prepared bundle, and configuration guides
- corrected the bundled Sonar agent skill to document only artifacts produced
  by the current prepared-bundle implementation

## Compatibility

- no HTTP routes, MCP tools, request or response models, prepared-bundle
  contracts, or retrieval policies changed
- the browser-capable image remains available through the `browser-runtime`
  target
- `bundle_version = 1` remains unchanged

## Verification

- `just check`: 100 passed, Ruff lint and formatting passed
- documentation links, tracked OpenAPI, and prepared-bundle artifact references
  passed repository-native checks
- default and MCP Compose configurations validated
- default Docker build verified as lightweight without Scrapling or
  CloakBrowser
- live HTTP health, search, and extraction passed
- live Streamable HTTP MCP health, search, and scrape passed
- OpenCode connected successfully to the Compose MCP endpoint
