# Sonar v0.1.0

Release date: 2026-04-08

## Summary

Sonar v0.1.0 is the first release of Sonar as a deterministic live-web evidence service for agent workflows.

This release establishes the core Sonar product:

- deterministic metasearch through SearxNG
- canonical low-level retrieval primitives for search, fetch, and extract
- thin HTTP/OpenAPI and MCP transport surfaces
- durable SQLite-backed caching for search and extracted documents
- high-level paper and source preparation flows for weaker local models
- durable prepared-source bundles for downstream indexing and note-writing

## Highlights

### Deterministic retrieval core

- normalized queries and generated stable query variants
- ranked and deduplicated results with domain priors and freshness-aware scoring
- cached search runs and fetched document records in SQLite

### Low-level composable APIs

- `sonar_search`
- `sonar_fetch`
- `sonar_extract`
- matching HTTP routes for local JSON clients

These remain the canonical composable Sonar surface.

### High-level research preparation

- `sonar_find_papers`
- `sonar_prepare_paper_set`
- `sonar_collect_sources_for_topic`

These flows collapse search, filtering, and extraction into fewer interactions for weaker local runtimes.

### Durable prepared-source bundles

- high-level preparation now returns a first-class `bundle` object
- bundles persist to disk by default as `prepared_source_bundle.json`
- optional `source_XX.txt` sidecars preserve extracted full text outside agent transcripts
- bundle provenance includes stable source IDs, search lineage, extraction method/status, cache flags, and request fingerprinting

### Multi-format extraction

Low-level extraction now supports:

- HTML
- PDF
- DOCX
- ODT
- Markdown
- plain text

This materially improves academic and document-heavy retrieval flows.

## Downstream Impact

Sonar is now a more reliable upstream component for systems like Synapse.

- prepared source evidence no longer depends on transcript retention
- persisted bundles are inspectable and auditable on disk
- downstream indexing and note-writing can target Sonar artifacts directly

## Known Limitation

Direct-document discovery from landing pages is still heuristic-based in this release. Current explicit coverage includes common academic patterns such as arXiv, OpenReview, ACL Anthology, and PMLR.
